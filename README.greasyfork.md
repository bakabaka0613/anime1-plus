# Anime1.me Plus

為 [anime1.me](https://anime1.me/) 打造的瀏覽器增強腳本：🖼️ 自動封面圖、⏯️ 觀看記錄與續播、🃏 海報卡片列表、📺 追番更新提醒、⛶ 網頁全螢幕、☁️ 多端同步。

> 📦 **本腳本開源於 GitHub** ── 原始碼、完整文件、問題回報與最新版本皆在
> **<https://github.com/bakabaka0613/anime1-plus>**
> 安裝後會自動從該 repo 檢查更新（`@updateURL`／`@downloadURL`）。

---

## 功能

### 封面圖（嚴謹匹配）
- 解析 anime1 標題（去集數/字幕組、辨識「第X季」「Ⅱ」「2nd Season」「劇場版」「OVA」等），到 [Bangumi](https://bgm.tv) 搜尋，用「名稱相似度 + 年份 + 季度」三維評分。
- **信心不足不靜默對錯**，改列候選讓你手選，選擇會記住。
- 封面卡主標用 **anime1 原始繁體名**（Bangumi 中文常為簡體），副標附 Bangumi 中／日文名。
- 動畫以穩定的 `categoryID` 為 key，分類頁／單集頁／列表頁共用同一份資料。
- **繁簡自動轉換（OpenCC）**：繁體標題轉簡體去搜（Bangumi 索引多為簡體）；對不上時再抓 Bangumi 別名比對救回。

### 觀看記錄 + 續播
- 在**分類頁就地內嵌播放器**看時（anime1 實際的觀看方式），自動用播放器的 `data-apireq` 精準辨識第幾話並記錄進度；單集頁 `/{postId}` 也支援。
- 重看自動跳回上次位置；確實播到 ≥90% 才標記「看完」（並防止播放器重載瞬間的 0 覆蓋進度）。
- 分類頁每集標示「已看 ✓ / 進度條」，頂部橫幅顯示進度：未看完一鍵**繼續看**、最後一集已看完則顯示**看下一集／看最新集**（跨分頁也能跳）。
- 分類頁多個內嵌播放器折疊成**上方選集列、下方單一播放器**，點集數即切換；進入時自動選**該續播的集**。特殊集在選集列顯示 **SP / OVA / OAD** 標籤（非「#」），分頁頁只剩一集也照樣顯示選集列。
- **單集頁導覽列**：底部整成一排按鈕 **`[上一集] 全集連結 [下一集]`**；上/下一集依和選集列相同的順序（數字集→特殊集）跨集跳，最後一個數字集也能接到 OVA/SP；無上/下一集時呈灰色不可點。
- **自動下一集**：看完自動切到下一集（折疊模式切選集列、單集頁倒數跳轉，倒數時可立即播放或取消）。

### 列表頁（海報卡片網格）
- 首頁 `/` 的動畫列表重排成**海報卡片網格**：封面在上、標題＋集數在下；封面 lazy + 限流 + 失敗重試載入。
- 封面比對信心不足時仍放圖，但加左上角「待確認」角標，誘導點進該動畫做嚴謹比對／手選。
- **Bangumi 評分**：海報右下角顯示 **★ 8.5**（無人評分則不顯示）。
- **更新提醒**：對有觀看記錄的動畫，若最新集數超過你已看完的集，卡片右上角顯示 **「+N」** 徽章。
- **懸浮工具列**（下滾黏頂）：搜尋框、卡片大小滑桿、**卡片 ⇄ 原始列表切換**。
- **下滾無限載入**：捲到底自動載入更多。點封面或標題都可進入該動畫。

### 播放體驗
- **網頁全螢幕**：播放器右上角明顯按鈕（亦可按 `W`），把播放器填滿整個視窗（非系統全螢幕）；播放中閒置時跟控制列一起自動淡出，滑鼠移入才顯示。
- **鍵盤快捷鍵**：`←/→` 快退/快轉（秒數可調、`Shift+←/→` 固定 10 秒）、`空白` 播放暫停（全域攔截）、`N` 下一集、`F` 系統全螢幕、`W` 網頁全螢幕、`+/-` 調速。
- **播放速度記憶**。

### 追番清單
- **浮動面板**（右下 📺）：列出在追的動畫（名稱用繁體）、一鍵繼續看；**分兩區排序**──上方「有進度」（可繼續看／看下一集／看新集），下方「已看完／已到最新進度」，各區內依最後觀看時間由新到舊。
- 已追平後又出新集者標「+N 新集」並歸在上方──只對「曾追到當時最新集」的動畫提醒，落後沒追完的不會跳。已看完最後一集者依連載狀態顯示「已到最新進度」或「已看完」；這兩種終端狀態也是**可點連結，回到你最後看的那一集**（低調灰色）。
- **管理模式**：**按住 Shift 點 📺**（或長按 📺 三秒，給觸控裝置）開啟，每部右側出現 **✓ 標記已看完** 與 **🗑 刪除進度**；平常開啟無這些鈕，避免誤觸。
- **🗑 軟刪除會同步**：清零進度並打刪除標記，會同步到其他裝置一起隱藏；**再次觀看即自動復原**。
- **封面預覽**：滑鼠移到列左側小封面縮圖即在面板旁浮出放大圖。

### 多端同步（GitHub Gist）
- 把**追番進度（觀看記錄 + 追番輔助資料）**同步到你自己的一個**私有 GitHub Gist**，多台裝置/瀏覽器共用。
- **只同步進度**：不含封面快取（各端各自向 Bangumi 重抓）與裝置設定（快進秒數、卡片寬度等各端獨立）。
- **逐集合併**：兩端看不同集會按時間戳逐集合併、互不覆蓋；流程為 `pull → 合併 → push`。
- **自動同步**：開啟動畫頁時先拉一次遠端；之後有新進度自動上傳。也可從油猴選單「☁️ 立即同步」。
- **設定**：油猴選單「☁️ 設定多端同步」→ 貼上只勾 Gist 讀寫權限的 GitHub Token；其他裝置貼**同一個 token** 即指向同一份。Token 只存本機、匯出 JSON 不含 token。
  👉 完整的 token 產生步驟、撤銷／更新說明見 [GitHub README](https://github.com/bakabaka0613/anime1-plus#多端同步github-gist)。

### 其他
- **右側欄隨檢視模式收放**：卡片檢視時自動隱藏右側欄（搜尋/近期更新等 widget），讓海報網格更寬；切回原始列表則顯示。
- **油猴選單**：JSON 匯出／匯入（合併模式）、「清除資料…」統一入口、開關各功能。

## 已知限制
- Bangumi 公開 API 有速率限制 → 結果快取、列表 lazy + 限流。
- 列表無限滾動需透過頁面的 DataTables 實例；拿不到時保留原生分頁。
- 播放器若改為非原生 `<video>`，續播/記錄會自動略過（不報錯）。
- 多端同步採 `pull→合併→push`（Gist 無鎖），對單人少數裝置足夠；單份同步資料上限 1MB。
